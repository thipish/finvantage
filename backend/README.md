# FinVantage ML Backend

## Quick Start

### 1. Install dependencies
```bash
cd backend
pip install -r requirements.txt
```

### 2. Set up MySQL
```bash
mysql -u root -p < sql/schema.sql
```

### 3. Train the model
```bash
python train_model.py
```
This generates `models/credit_risk_model.pkl` and supporting artefacts.

### 4. Configure environment variables
```bash
# Required — use a strong, unique password
export DATABASE_URL="mysql+pymysql://your_user:YOUR_STRONG_PASSWORD@localhost:3306/finvantage"

# Required — generate a random API key for frontend-backend auth
export FINVANTAGE_API_KEY="$(openssl rand -hex 32)"

# Optional — comma-separated list of allowed frontend origins
export ALLOWED_ORIGINS="http://localhost:5173,https://finvantage.lovable.app"
```

### 5. Start the API server
```bash
uvicorn app:app --reload --port 8000
```

### 6. Test the endpoint
```bash
curl -X POST http://localhost:8000/api/predict \
  -H "Content-Type: application/json" \
  -d '{
    "fullName": "Rahul Sharma",
    "age": 30,
    "dependents": 1,
    "maritalStatus": "Married",
    "employmentLength": "3-5",
    "homeOwnership": "RENT",
    "cibilScore": 720,
    "annualIncome": 800000,
    "monthlyDebt": 12000,
    "totalInvestments": 200000,
    "bankBalance": 50000,
    "loanAmount": 1500000,
    "loanPurpose": "Home",
    "interestRate": 11.5,
    "loanTermMonths": 60
  }'
```

## Architecture
```
React Frontend → POST /api/predict → FastAPI → Random Forest .pkl → MySQL → Response
```

## Files
- `sql/schema.sql` — MySQL table definitions
- `data/sample_historical_loan_data.csv` — Training dataset
- `train_model.py` — Model training script (Random Forest)
- `app.py` — FastAPI server with SQLAlchemy ORM
- `models/` — Exported .pkl artefacts (generated by train_model.py)
